# TAM 2025 Schedule — Project Overview

## Purpose

This project will be a single-page schedule site for the TAM (Texas Association of Museums). It presents daily schedule blocks with session details, speakers, locations, and optional links. It will also provide a login functionality for admins that will allow them to perform CRUD operations, change the color scheme, add announcements, and possibly other features as the app evolves.

## Key Features

- **Schedule-driven UI**: The day-by-day agenda is defined in `accordionProps` and rendered into expandable sections.
- **Session details**: Each session can include time, location, sponsor, description, speaker list, and optional registration or slide links.
- **Notifications banner**: Dismissible announcements are displayed at the top of the schedule. they can be success/warning/error type banner depending on what admin sets.
- **Accessible navigation**: A top navigation bar provides primary routing.
- **Footer links**: Official links are centralized and rendered in the footer.
- **Admin login**: login button at top right of app that allows admins to login.
- **Admin section**: Perform CRUD operations on events and event items. In this section, each event item will have a clickable edit item that opens the admin CRUD modal.
- **Admin CRUD modal**: Modal that performs CRUD operations on events and event items.
- **Admin color scheme picker**: Modal that opens to update the colorscheme (more on this to come.)
- **Admin announcement functionality**: modal that will appear for admins to add type of, name, description of announcements.
- **Client build**: Created using Vite for bundling and development tooling.

## Routes

- **Home layout**: Renders the hero, notifications, navigation, the active child route, and the footer.
- **Welcome (index)**: Displays the accordion schedule by day.
- **Conference sponsors**: A standalone sponsors page (static)
- **Multi-event view**: Users can add/select more than one event; the route passes those event IDs to the backend, which returns the specific event data to render.

## Components

- `Accordion`: Expand/collapse container for a single day of events; renders `EventComponent` for each session.
- `Cancelled`: Simple “Cancelled” label used inside events.
- `EventComponent`: Session card showing time, title, sponsor, location, links, description, and speaker headshots.
- `Footer`: Renders official outbound links from configuration.
- `Headshot`: Displays speaker name and institution with placeholder imagery.
- `Hero`: Page title with TAM logo.
- `NavBar`: Route links with active state styling.
- `Notifications`: Dismissible announcement banner.
- `OutboundLink`: Reusable external link with optional dividers.

## Styling

- styling will be handled by scss (needs to be configured). we are going to create our own styling library through sass mixins, etc. these need to be responsive based off what the backend style hex codes are.
- styling hexcode structure (for now) is:
  ```
  {
      primary: str
      secondary: str
      tertiary: str
      background: str
      alt_background: str
      text: str
      title_text: str
  }
  ```

## Data Sources

- **Schedule content**: fastAPI backend located at `localhost:8000` for local dev. Will update w/ env vars once backend server is up and running for prod so it will dynamically set the base route off this.
- **Speaker data (generated)**: `app/constants-v2.ts` is generated by `extract-data.js` using the CSV file at the project root.
